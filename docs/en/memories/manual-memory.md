# How to Save a Memory Manually

You do not have to wait for auto capture.

If you already know that something is important and should be remembered right now, you can save it manually.

## When manual save is useful

- You just confirmed an important conclusion
- You found a bug fix that is easy to forget
- The information is short but highly valuable
- You do not want to wait for automatic capture

## How to do it in the WebUI

At the bottom of the page, there is an “Add New Memory” form.

Fill it in like this:

1. Choose a tag
2. Choose a type
3. Add extra tags if needed
4. Write the content
5. Click “Add Memory”

## What each field means

### Tag

This decides where the memory belongs.

### Type

This helps organize memories by purpose, such as feature, bug fix, architecture, rule, or documentation.

### Extra tags

Use short keywords that will help future search.

### Content

Write the thing you actually want to remember.

The best manual memories are clear, specific, and reusable.

## Is this the same as “force add”?

You can think of it that way.

Auto capture lets the plugin decide what to save. Manual add means you are explicitly telling it: save this now.

## Is there another active save method?

In some environments, the memory tool can also add content directly.

But for most users, the WebUI form is the easiest and most reliable way to do it.
