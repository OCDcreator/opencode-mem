# How to Browse and Search Memories

This part of the plugin helps you recover useful past context quickly.

## What you can do in the WebUI

- Browse saved project memories
- Search past content by keyword
- Filter results with tags
- See whether a memory is linked to an original prompt
- Pin important memories for quick access

## The two things you usually see

### Memories

These are the cleaned-up results created by the plugin.

They are usually shorter and easier to reuse than the original conversation.

### Prompts

These are the original user messages that started the interaction.

Sometimes a prompt and its generated memory are shown together, so you can see both the question and the outcome.

## Common ways to use search

### Recall project rules

Search by module name, feature name, or a team convention to recover earlier decisions.

### Revisit old bug fixes

If you think “we solved this before”, search by error text or feature name and check the older memory.

### Prepare for a new session

Before starting a fresh task, search for the project area you are about to touch. That often saves a lot of repeated explanation.

## How to search better

- Start with business terms, module names, or error messages
- If nothing appears, try a shorter keyword
- Use tags to narrow things down when needed

The goal is not to reread your full history. The goal is to find the one useful memory quickly.
