# What the User Profile Page Is For

The User Profile page is not about the project itself.

It is about your long-term working habits when you collaborate with AI.

## What you may see there

### Preferences

Examples:

- You prefer concise answers
- You tend to like a certain code style
- You often want explanation before implementation

### Patterns

Examples:

- Topics you return to often
- Problem types you ask about repeatedly
- Technical areas that show up again and again

### Workflows

Examples:

- You usually ask for analysis first, then implementation, then testing
- You often debug first and refactor after that

## Why this page matters

- It helps you understand how the plugin sees your working style
- It gives the system a better chance to align with your habits over time
- It shows whether profile learning is actually happening

## When it updates

It does not update after every single prompt.

Usually the plugin waits until enough usage history has built up before running a new analysis.

So if the page is empty right after setup, that is normal.
