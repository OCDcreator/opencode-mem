# OpenCode Memory 是做什么的

OpenCode Memory 是一个给 OpenCode 用的长期记忆插件。

它的作用很简单：把你和 AI 协作过程中真正有用的内容留下来，等你以后再回到这个项目时，可以更快找回背景、结论和已经验证过的做法。

## 这个插件能帮你做什么

- 自动整理有价值的对话内容，生成更适合长期保存的记忆
- 让你在 WebUI 里搜索、筛选、查看项目记忆
- 支持你手动保存一条“现在就想记住”的内容
- 展示用户画像，帮助你了解自己的偏好和工作习惯
- 通过配置 Provider 和嵌入模型，让自动采集和搜索工作起来
- 同时支持远端嵌入和本地嵌入模型两种搜索方案
- 支持全局配置，也支持项目级配置覆盖

## 最推荐先看的页面

- **项目记忆**：先了解记忆列表里都有什么、怎么看、怎么搜
- **手动保存记忆**：如果你不想等自动采集，可以直接手动存
- **自动采集**：了解插件会在什么时候替你自动保存
- **配置向导**：第一次配置时最值得先看

## 左侧每一类都在讲什么

### 使用与浏览

- **项目记忆**：项目记忆页里有哪些内容、怎么搜索、怎么筛选、怎么置顶
- **手动保存记忆**：如何主动添加一条记忆，等同于“强制记住这件事”
- **自动采集**：插件什么时候会自动保存内容
- **用户画像**：用户画像里有哪些信息、怎么理解这些信息
- **存储与检索**：如何更高效地找回过去的记忆

### 配置与模型

- **配置向导**：从零开始把插件配置好的最短路径
- **嵌入模式**：搜索效果依赖什么模型、该怎么选
- **Provider 配置**：自动采集和画像分析用哪个模型、该怎么填

### 排查与诊断

- **日志与调试**：功能没生效时，最先应该去哪里看

## 如果你是第一次使用

建议按这个顺序看：

1. 配置向导
2. Provider 配置
3. 项目记忆
4. 自动采集

这样你会最快知道这个插件怎么配、配好后会出现什么、以及平时怎么用。
