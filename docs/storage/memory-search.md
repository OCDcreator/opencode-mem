# 你能怎样查看和搜索记忆

这个页面最实用的价值，是让你把“以前聊过但记不清了”的东西找回来。

## 在 WebUI 里你能做什么

- 看项目里已经保存过的记忆
- 用关键词搜索历史内容
- 按标签筛选某一类记忆
- 查看某条记忆是不是和某个提示词关联
- 置顶重要内容，方便后面快速回看

## 你看到的内容大概分两种

### 记忆

这是插件整理过后的结果。

一般更短、更容易复用，也更适合以后搜索。

### 提示词

这是当时你真正发给 AI 的那句话。

有些情况下，提示词会和生成出来的记忆成对显示。这样你不仅知道“结论是什么”，也知道“当时是怎么问出来的”。

## 最常见的使用方式

### 回忆项目规则

搜索某个模块名、文件名、约定名，就能快速找回之前形成的结论。

### 排查重复问题

当你感觉“这个 bug 好像以前修过”，直接搜报错关键词或相关功能名，往往能找到历史处理记录。

### 给新会话补上下文

在重新开一个新任务前，先搜一下这个项目的核心功能、约定和坑点，可以少走很多重复沟通。

## 怎样搜更容易找到

- 优先搜业务词、模块名、错误信息
- 搜不到时，换一个更短的关键词
- 再不行就结合标签筛选

搜索的目标不是把整段历史翻出来，而是尽快找到“哪条记忆值得点进去再看”。
