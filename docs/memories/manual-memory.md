# 怎么手动保存一条记忆

除了等自动采集，你也可以主动把一条内容保存下来。

如果你心里想的是“这件事我现在就要记住，不想等自动判断”，那你要用的就是这个功能。

## 什么时候适合手动保存

- 你刚确认了一个重要结论
- 你发现了一个很容易再踩的坑
- 这条信息很短，但非常值得以后再看
- 你不想等自动采集

## 在 WebUI 里怎么操作

页面下方有一个“添加新记忆”区域。

你只需要按顺序填写：

1. 选择标签
2. 选择类型
3. 填写补充标签（可选）
4. 输入内容
5. 点击“添加记忆”

## 每个输入框是做什么的

### 标签

决定这条记忆属于哪个项目或容器。

如果不选标签，这条记忆不知道该归到哪里去。

### 类型

帮助你把记忆分成不同用途，比如：

- 功能
- 修复
- 架构
- 规则
- 文档

### 补充标签

适合填关键词，方便以后搜索。

比如：

`react, auth, sqlite, migration`

### 内容

这里写你真正想保存的东西。

最好的写法是：

- 直接写结论
- 写清楚适用场景
- 必要时补一句原因

## 一个好例子

“项目里的用户资料接口不能直接删字段，前端旧版本还在依赖 `displayName`，需要先做兼容层。”

这种内容以后再搜时会非常有价值。

## 一个不太好的例子

“今天看了一下这个功能，感觉有点问题。”

这种内容太模糊，以后看到也很难直接用。

## 这是不是“强制添加”

可以这样理解。

自动采集是插件自己判断要不要保存；手动添加则是你明确告诉插件：“这条内容请现在就记住。”

## 还有没有别的主动保存方式

在一些支持工具调用的环境里，也可以通过插件暴露的记忆工具主动执行添加。

但对大多数用户来说，最直接、最稳定的方式还是 WebUI 里的手动添加表单。
