# 项目记忆页怎么用

“项目记忆”是这个插件最常用的页面。

你可以把它理解成：这个项目的历史经验库。

## 你会在这里看到什么

### 记忆

这是插件整理后的内容。

通常会比原始聊天更短，也更适合以后直接复用。

### 提示词

这是你当时真正发给 AI 的提问。

有些情况下，提示词和记忆会连在一起显示，这样你既知道结论，也知道当时的问题是什么。

## 你能在这里做什么

- 搜索历史记忆
- 按标签筛选
- 查看某条记忆的详细内容
- 置顶重要记忆
- 编辑记忆
- 删除单条或批量删除

## 最常见的用法

### 想回忆之前怎么解决过某个问题

直接搜索报错关键词、模块名、功能名。

### 想知道这个项目有哪些固定规则

搜项目约定、命名方式、目录结构、接口名等关键词。

### 想把重要内容一直放前面

可以把它置顶，后面再看会方便很多。

## 列表里不同内容代表什么

- 带有更完整总结的内容，通常是插件整理过的记忆
- 带有原始上下文感的内容，通常更接近当时的提示词
- 两者连在一起时，说明这条记忆和原问题已经关联起来了

## 什么时候这个页面最有价值

- 重新打开一个老项目的时候
- 感觉“这个问题以前好像遇到过”的时候
- 想给下一次 AI 协作补上下文的时候

如果你只打算学会一个页面，优先把这个页面用熟。
